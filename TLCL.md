# regex

literal characters: characters that *match themselves*
meta characters: to specify more complex matches: `^ $ . [ ] { } - ? * + ( ) | \`
`\`: used to escape literals to meta or escape meta to be literal.

## any character

`.` matches any character in that character position.
`'.zip'`:  anything at the start and end

## anchor

`^` (caret) start and `$` end of the line.
`'^$'`: will match blank lines
`'^zip$`: the ones with only `zip`.

## bracket expression

in brackets, meta characters are considered literals

`[bg]zip`: match one from a list of characters.
`'[bg]zip'`: only `b` or `g` before zip.
`[^bg]zip`: negation; without these characters at the given position.

## character ranges

to specify sets of characters

`^[A-Z]`: starts with uppercase
`^[a-Za-z0-9]`: matches everything that starts with letters and numbers
`[A-Z]`: every filename with an uppercase letter
`[-AZ]`: every file name that has either of these.

## POSIX character classes

another way of specifying ranges.

ASCII collation order: `ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz`
dictionary order: `aAbBcCdDeEfFgGhHiIjJkKlLmMnNoOpPqQrRsStTuUvVwWxXyYzZ`

in locale based settings, the POSIX-compliant apps use a dictionary order. so [A-Z] in
dictionary order is all but character `a`.

POSIX standard includes a number of character classes that provide useful ranges of
characters:

`[:alnum:]`: [A-Za-z0-9]
`[:alpha:]`: [A-Za-z]
`[:space:]`: The whitespace characters including space, tab, carriage return, newline and ...
`[:digit:]`: The numerals 0 through 9.
`[:blank:]`: Includes the space and tab characters

## POSIX basic and extended regex (BRE vs ERE)

they differ in meta character:

BRE: `^ $ . [ ] *`
ERE: `( ) { } ? + |`

* `( ) { }` will be *meta* if escaped in BRE.
* any ERE meta characters will be literal if escaped.

all the rest need `-E`

## quantifiers

the number of times something is matched

### `?`: makes the element optional. zero or once

`^\(?[0-9][0-9][0-9]\)?`: `( )` are optional and also escaped to be literal.

### `*`: zero or more

`[[:upper:][:lower:] ]*\.`: as much upper or lower case or spaces and ends with a literal
period.

### `+`: at least one

`^([[:alpha:]]+ ?)+$`: starts with one or more alphabetical character, one space or not, and
at least one of these.

### `{ }`: match a specific number of times

`{n}`: match if occurred n times
`{n,m}`: at least n, but not more than m
`{n,}`: n or more
`{,m}`: no more than m

`^\(?[0-9]{3}\)?`: match 3 times
